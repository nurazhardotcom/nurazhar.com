Debugging Hermes Agent's Memory Character Limit

The Error

I was updating my user profile in Hermes Agent's memory. Simple task — add my career trajectory. The tool rejected it:

Replacement would put memory at 1,467/1,375 chars.
Shorten the new content, or 'remove' other stale or less important
entries to make room, then retry.

I shortened the text. Retried. Same error. Again. The tool got stuck in a loop — three identical failures because the user profile store was at capacity and I was adding without removing.

The Architecture

Hermes Agent has two memory stores:

StorePurposeLimit
userWho you are — name, preferences, background1,375 chars
memoryAgent notes — environment, conventions, lessons2,200 chars

Every entry gets injected into the system prompt on every single turn. At ~4 chars per token, the user profile costs roughly 344 tokens per turn. The agent notes cost ~550. Combined: ~900 tokens of overhead before any conversation history or tool output enters the prompt.

What Went Wrong

My user profile had two entries with overlapping career information:

Entry 1 (993 chars):

Singapore, CachyOS. Born 1986 (40), INTJ. 7yr IT ops ICA (CyberArk, Entra ID, Carbon Black, Tenable, Guardium). Unemployed 1yr. Debt $40k SGD. Active: lagu-lagu...

Entry 2 (367 chars):

Considering a career pivot/return to tech workforce after 1yr unemployment. Has 7yr IT ops background (CyberArk, Entra ID, Carbon Black, Tenable, Guardium) at ICA. Currently pursuing Professional Diploma...

Total: 1,360/1,375 chars. Same career details duplicated across both entries. No room for anything new.

The Fix

Step 1: Remove the old entry (Entry 1).

memory action=remove target=user old_text="Singapore, CachyOS. Born 1986..."

Step 2: Write a consolidated entry:

Singapore, CachyOS. Born 1986 (40), INTJ. Career: IT Support → Cybersecurity Analyst → Compliance/TechFresh → Application Support (DCS, Power Automate/Odoo/WordPress) → now Full Stack Dev (Lithan diploma + lagu-lagu). 7yr total. Unemployed 1yr. Debt $40k SGD. Post-Muslim. Direct, no soft-pedal. Voice via KDE. Anti-bloat.

Result: 1,318/1,375 chars. Fits. No duplication.

Why the Limit Exists

ReasonDetail
Context windowEvery memory token competes with conversation tokens
Token cost~900 tokens of overhead per turn, multiplied across thousands of calls
Signal-to-noiseForces you to keep only stable facts, not task logs
PerformanceCompact memory = faster turns

Lessons

  1. Memory is not a log. It's a compact identity document. Facts, not narratives.
  2. Deduplicate aggressively. Same info in two entries = wasted space.
  3. Declarative beats verbose. "Career: A → B → C" beats three paragraphs.
  4. Prune stale entries. The old entry was from before I understood my own trajectory.
  5. The limit is a feature. Constraints force better decisions.

Quick Reference
